CapsLock 三态切换

关闭 / 兼容（短按切换）/ 纯切换（按下即切换 + IOKit 钳制 LED，完全禁用大写锁定）。纯切换模式与 macOS 原生「⇪ 切换 ABC 输入源」互斥，启用前 App 会弹窗引导。

非"关闭"模式需要辅助功能权限

技术实现

TIS 乐观缓存

切换成功后直接写入目标状态，避免 TIS 异步生效期间的缓存错误，彻底修复切换失效问题。

CGEvent tap 拦截

在会话层捕获 CapsLock flagsChanged 事件，短按（< 300ms）直接切换输入法；tap 创建成功即作为 AX 权限依据，绕开进程内缓存问题。

事件合并去重

NSWorkspace 激活通知经 8ms DispatchWorkItem 合并，过滤同 App 重复事件，快速 Cmd+Tab 更低延迟。

权限双档轮询

AccessibilityManager 等待授权期间以 1s 高频轮询，常态切换为 5s 低频，授权后自动恢复拦截器无需重启。

懒构建菜单

NSMenuDelegate 实现菜单内容仅在点击时渲染，消除每次 App 切换时的 UI 主线程压力。

全量应用发现

mdfind Spotlight 扫描 + 递归深度遍历双通道，以 path: 合成 Bundle ID 支持无 Info.plist 应用（游戏客户端等）；NSOpenPanel 兜底手动选取任意 .app。

标准菜单栏注入

以 .regular 激活策略运行，AppDelegate 动态注入 App / Edit / Window 菜单，确保 Cmd+W/A/C/V/Z 等系统快捷键在设置窗口内正常工作。

代数守卫 re-read

switchGeneration 计数器确保只有最新一次 TIS 切换的延迟补读能清除 pendingSwitchTarget，彻底解决设置了应用规则后纯切换模式失效的问题。

中文子模式直选

切换为中文时优先通过 kTISTypeKeyboardInputMode 子节点直接激活中文 submode，而非仅选父级 source。解决 Apple Pinyin 在英文 submode（SCIM.ABC）下切回中文后 HUD 显示正确但实际仍输入英文的问题。
